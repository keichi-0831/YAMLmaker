## 项目简介

本项目是一个纯前端的人设卡辅助工具，核心目标是：

- 把零散的人设想法整理成结构化表单
- 让角色卡、世界书、灵感池、AI 辅助放在同一个工作流中
- 降低 YAML / 世界书手动排版成本
- 同时兼顾 **本地缓存使用** 与 **云同步备份**


---

## 核心功能

### 1. 人设卡编辑

支持从多个维度完整填写角色信息，包括但不限于：

- 基础信息
  - 角色名 / 中文名 / 昵称
  - 年龄 / 生日 / 星座 / 性别 / 身高
  - 身份标签 / 人设类型 / 社会评价
- 背景故事
  - 童年 / 少年 / 青年 / 现状
  - 每个阶段都可以填写多条经历
- 外貌穿着
  - 发型、眼睛、皮肤、脸型、体型
  - 正装、商务休闲、日常、居家穿着
- 性格特质
  - 核心性格、恋爱性格、弱点、喜恶、人生目标
- 与 `{{user}}` 的关系
  - 共同经历、印象、关系备注
- 行为习惯
  - 生活习惯、工作习惯、情绪表现、关系边界、技能特长
- 说话风格
  - 对 `{{user}}` 的语气、风格逻辑、口音、线上聊天风格
- 额外补充
  - 口头禅、小动作、创伤、信念、矛盾、秘密、人际关系、关键记忆
- NSFW 信息
  - 性经验、性取向、定位、习惯、Kinks、Limits

### 2. YAML 导出

可直接将人设卡导出为结构化 YAML，支持：

- 预览 YAML
- 一键复制 YAML 到剪贴板
- 自动跳过空字段
- 自动将自定义代称替换为 `{{user}}`

适合后续投喂到聊天模型、角色平台、笔记系统或自行二次编辑。

### 3. 世界书编辑与导出

除了角色本体，本项目还支持维护与角色相关的世界观信息，包括：

- 状态栏
- 时代背景
- 特殊设定
- NPCs
- 前端美化需求
- 人设纠偏
- 指定状态
- 额外补充

并支持两种导出方式：

- **YAML 复制导出**
- **SillyTavern 世界书 JSON 导出**

其中 JSON 导出部分带有：

- 条目名称
- 触发方式（常量 / 关键词）
- 关键字
- 插入位置
- 插入深度
- 插入顺序

这意味着它不仅能“写设定”，还能更直接地接到世界书工作流里使用。

### 4. 灵感池

用于收集碎片化灵感词，帮助你在创作前快速抓氛围。

支持：

- 手动添加灵感词
- 批量导入
- 上传 `.txt` 文件导入
- 复制整个词库
- 清空词库
- 随机抽取若干词条作为“灵感时刻”

适合用于：

- 想设定但没想法时破冰
- 给 AI 一个更聚焦的创作方向
- 快速拼出关系感、情绪感、剧情感

### 5. 懒人起名器

内置随机起名功能，支持：

- 姓氏池
- 名字字池
- 默认词池载入
- 批量导入
- 复制词池
- 生成二字名 / 三字名
- 批量生成多个候选名字
- 可选“不叠字”

适合在角色初建阶段快速得到一组可用名字候选。

### 6. AI 辅助生成

AI 模块是本工具的重点之一，包含：

- API 供应商管理
- API 地址 / Key 配置
- 模型列表获取
- 流式输出
- 原始回复查看
- 解析后的结果展示
- 一键填回表单

并且 AI 生成不是“整段自由输出”，而是按模块组织的。

#### 支持的 AI 创作方向

- 人设卡生成
- 世界观 / 世界书补全
- user 人设生成
- 开场白生成

#### 支持的 AI 输入上下文

- 系统 Persona / 创作身份定义
- 本次灵感词
- XP_core（你这次想要什么）
- 已填写的人设卡内容
- 已填写的世界书内容
- 起名池内容
- 自定义引导词
- 开场白人称偏好
- 抢话 / 不抢话设置
- 字数范围

#### AI 模块化引导

用户可以单独勾选是否让 AI 生成某一模块，例如：

- 基础信息
- 背景故事
- 外貌穿着
- 性格特质
- `{{user}}` 相关
- 行为习惯
- 说话风格
- 额外补充
- NSFW

世界观与 user 人设、开场白也有对应的独立引导区，整体非常适合“半自动创作”。

### 7. 角色档案管理

项目支持多角色存档：

- 新建角色
- 重命名角色
- 删除角色
- 切换角色
- 自动保存当前编辑内容

适合同时维护多个 OC 或不同企划角色。

### 8. 云同步

项目支持通过 **同步 ID** 进行云端同步，功能包括：

- 新增同步 ID
- 删除同步 ID
- 切换同步 ID
- 从云端恢复到本地
- 将本地数据保存上传到云端

同时会同步：

- 角色数据
- 灵感池
- 起名池
- AI 配置中的大部分创作配置

> 注意：从代码逻辑看，**API 地址和 API Key 不会上传到云端**，只保留在本地浏览器缓存中。

## 使用流程建议

如果你是第一次使用，可以参考下面这个顺序：

1. 打开网页
2. 新建一个角色档案
3. 在“基础信息”到“额外补充”中逐步填写角色
4. 如果需要长线剧情 / 世界观支持，再补“世界书”
5. 没灵感时去“灵感池”Roll 词
6. 需要名字时用“懒人起名器”快速生成
7. 想偷懒或扩写时，进入“AI 辅助”配置模型并生成
8. 最后导出 YAML 或世界书 JSON 使用
9. 如果有跨设备需求，再配置同步 ID 并手动上传 / 恢复

## 界面结构概览

页面主要分为四个顶级标签：

- **📄 人设卡**：角色本体信息编辑与 YAML 导出
- **🌍 世界书**：世界观信息维护与世界书导出
- **✨ 灵感池**：灵感词管理与随机抽取
- **🤖 AI 辅助**：模型配置、提示词构建与结果回填

此外，页面顶部还有两个非常重要的管理栏：

- **云同步 ID 管理栏**
- **角色档案管理栏**

这使得整个应用更接近一个小型“创作面板”，而非单次输入表单。

## 数据存储说明

本项目以浏览器本地存储为基础：

- `localStorage` 用于保存角色数据
- `localStorage` 用于保存当前角色 ID
- `localStorage` 用于保存灵感池、起名池、同步 ID 等配置

默认情况下：

- **不需要登录**
- **不需要服务器即可本地使用**
- **刷新页面后数据仍会保留在当前浏览器中**

若启用云同步，则通过 Supabase 将数据上传到云端表中进行恢复与同步。

## 云同步安全提醒

项目界面中已经明确提醒，重点说明：

- 同步 ID 是访问云端数据的唯一凭证
- 请使用复杂字符串，不要使用简单单词或短数字
- 建议混合：**大小写字母 + 数字 + 符号，长度 20 位以上**
- 一旦同步 ID 被他人猜到，对方可能访问到你的云端数据

如果你不需要云同步，可以完全忽略该功能，仅使用本地缓存模式。

## 使用注意事项

### 1. 本地缓存依赖浏览器

如果你：

- 清空浏览器缓存
- 更换浏览器
- 使用无痕模式

那么本地数据可能无法保留或无法互通。

### 2. 云同步是手动上传 / 恢复思路

从代码逻辑来看，云同步更偏向：

- 本地先编辑
- 需要时手动“保存上传”
- 需要时手动“恢复同步”

因此建议养成手动上传的习惯，避免跨设备时出现版本差异。

### 3. API Key 仅本地保存

这对安全是好事，但也意味着：

- 换浏览器 / 换设备后需要重新填写 API 配置
